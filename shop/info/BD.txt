1. `orders` - заказы пользователь (купоны):
				id - номер заказ
				user_id	- номер пользователя
				coupon	- номер купона
				data - дата заказа
				email - эмейл на который отправлять купон
				product_id - номер товара

2. `orders_maps` - заказы малого бизнеса (подписки):
				id - номер заказ
				user_b_id - номер пользователя (владельца малого бизнеса)
				quantity - количество месяцев подписки
				data - дата заказа
				data_start - дата появления точки на карте
				phone - телефон для связи 
				adress	- адресс добавления на карту (подтвержденный документально)
				status	- статус (новый, на расмотрении, добавлен на сайт, завершен)
				data_stop - дата окончания подписки (снятия гео-метри с карты)

3. `products` - продукты по купону:
				id	- номер товара
				title	- название
				cost - цена	
				description	- краткое описание
				content	- описание
				image	- картинка
				category_id	- категория

4. `products_maps` - подписки:
				id	- номер товара
				title	- название
				cost - цена	
				content	- описание
				image	- картинка

5. `request` - заявки на добавление продуктов (купонов):
				id	- номер товара
				title	- название
				cost - цена	
				description	- краткое описание
				content	- описание
				image	- картинка
				category_id	- категория
				user_id	- пользователь (предприниматель) желающий разместить купон
				phone	- его телефон для связи
				status - статус заявки (новый, на расмотрении, добавлен)

6. `users` - пользователи:
				id	- номер пользователя
				login	- его логин
				password	- его пароль
				email	- его почта для верификации
				name	- его имя
				phone	- его телефон
				confirm_mail	- код верификации полученый пользователем на почту
				verifided - прошел ли пользователь верификацию (если 1, то да, если 0, то нет)

7. `users_b` - пользователи (владельцы малого бизнеса):
				id	- номер пользователя
				login	- его логин
				password	- его пароль
				email	- его почта для верификации
				name	- его имя
				phone	- его телефон
				confirm_mail	- код верификации полученый пользователем на почту
				verifided - прошел ли пользователь верификацию (если 1, то да, если 0, то нет)
				info	- документация подтверждающая что это действительно малый бизнес
